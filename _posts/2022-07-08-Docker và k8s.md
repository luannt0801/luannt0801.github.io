### 1. Linux file hierarchy

Linux tổ chức hệ thống file theo dạng cây, bắt đầu từ thư mục gốc `/`. Tất cả file và thư mục đều nằm dưới root này. Một số thư mục quan trọng:

* `/bin`: chứa các lệnh cơ bản như `ls`, `cp`, `mv`
* `/etc`: chứa file cấu hình hệ thống
* `/home`: thư mục người dùng
* `/var`: dữ liệu thay đổi thường xuyên (log, cache)
* `/usr`: chứa chương trình và thư viện
* `/tmp`: file tạm thời

Cách tổ chức này giúp hệ thống rõ ràng, dễ quản lý và phân quyền.

---

### 2. Docker

Docker là nền tảng giúp đóng gói ứng dụng thành container để đảm bảo chạy nhất quán trên mọi môi trường.

#### Dockerfile và Docker Compose

* Dockerfile dùng để build image
* Docker Compose dùng để chạy nhiều container và định nghĩa cách chúng kết nối với nhau

Dockerfile mô tả cách tạo image. Docker Compose mô tả cách chạy hệ thống nhiều service.

---

### Các file quan trọng trong Docker

#### `.dockerignore`

Dùng để loại bỏ các file không cần thiết khi build image.

Ví dụ:

```plaintext
node_modules
.git
*.log
.env
dist
```

Lợi ích:

* giảm kích thước build context
* tăng tốc độ build
* giảm kích thước image

---

#### `docker-compose.override.yml`

File này được Docker Compose tự động merge với file chính.

Use case:

* file chính dùng cho production
* file override dùng cho môi trường local

---

#### `docker-compose.prod.yml`

Dùng riêng cho production.

Ví dụ:

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml up
```

---

#### `.env`

Chứa biến môi trường dùng trong Docker Compose.

Ví dụ:

```plaintext
DB_USER=admin
DB_PASSWORD=123456
PORT=3000
```

---

#### `Dockerfile.dev`

Một project có thể có nhiều Dockerfile:

* `Dockerfile`: mặc định
* `Dockerfile.dev`: cho development
* `Dockerfile.prod`: cho production

---

#### `docker-entrypoint.sh`

Script chạy trước khi container khởi động.

Ví dụ:

```bash
#!/bin/sh

npm migrate
npm start
```

### 2. Docker Network

Docker tạo một lớp mạng ảo để các container giao tiếp với nhau.

#### Các loại network chính

* bridge
* host
* overlay

---

#### Bridge network

Đây là network mặc định.

Khi chạy container:

```bash
docker run nginx
```

Container sẽ được attach vào bridge network (docker0).

Mỗi container có:

* IP riêng
* network namespace
* virtual ethernet

Ví dụ IP:

```plaintext
172.17.0.2
172.17.0.3
```

---

#### Docker Compose network

Docker Compose tự tạo network riêng cho mỗi project.

Ví dụ:

```plaintext
project_default
```

Các container trong cùng compose sẽ nằm trong cùng network này.

---

#### DNS nội bộ

Docker cung cấp DNS nội bộ.

Container có thể gọi nhau bằng service name thay vì IP.

Ví dụ:

```plaintext
backend → db:5432
```

---

### 3. Kết nối service trong container

#### Bản chất container

Container là một process Linux được cô lập bằng namespace.

Ví dụ một server chạy trong container:

```javascript
app.listen(3000)
```

---

#### Container kết nối container

Ví dụ backend kết nối database:

```plaintext
postgres://db:5432
```

Luồng hoạt động:

* backend gửi request
* DNS nội bộ resolve tên `db`
* kết nối tới container database

---

#### Container kết nối host

Container có thể gọi service trên host thông qua:

```plaintext
host.docker.internal
```

Ví dụ:

```plaintext
http://host.docker.internal:8080
```

---

#### Host kết nối container

Cần publish port:

```bash
docker run -p 8080:80 nginx
```

Luồng:

* client gọi `localhost:8080`
* host forward tới container port 80
* process trong container xử lý

---

### Luồng network tổng thể

```plaintext
Internet
   │
Host machine
   │
Port mapping
   │
Container
   │
Service process
   │
Docker network
   │
Các container khác
```

---

### Ví dụ microservice với Docker Compose

```yaml
services:

  api:
    build: .
    ports:
      - "3000:3000"

  db:
    image: postgres

  redis:
    image: redis
```

Network:

```plaintext
compose_network
 │
 ├── api
 ├── db
 └── redis
```

Kết nối:

```plaintext
api → db:5432
api → redis:6379
browser → localhost:3000
```
